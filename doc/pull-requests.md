# Development Workflow & Branching Strategy

Each branch in this project is **module/package-specific**.  
- **Module branches** are used for active development (code pushing).
- **Package/main branches** are PR-merged and therefore **protected**.

---

## üîß Development Process

1. **Open an issue**
2. **Create package branch(es)** associated with the issue
3. **Create module branch(es)** associated with a package branch
4. **Implement features** on the module branch(es)
5. **Submit a PR** from each finished module to its corresponding package branch
6. **Submit a final PR** to `main` once *all* packages for the issue are complete
   - Merge  
   - Close the issue  
   - Delete unused branches
